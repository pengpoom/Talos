## When to Use
- **排未来**：用户说"帮我把今天排进日历 / 给我占个块写 X / 安排下午的专注时间 / 把这几件排进日程"。
- **补录实际**：用户说"我刚 10-11 写了 lunwen2 / 刚才做了 Z 帮我补上"——把已经发生的活补到日历 + 时间轴。
- **不限领域**——科研、工作、个人的事都能排（放在 research/ 只是归档，不限制调用）。

## Procedure

### A. 排未来（规划，把今天要做的排进日历）

1. **拿到要排的事**：用户直说的；或刚 `morning-plan` 提的 1-3 件；或飞书 Todo 里今天的。每件估个时长（没说默认 **50 分钟/块**）。

2. **查忙闲（关键，别撞车）**：用 `lark-calendar` 的 **freebusy / 推荐空闲时段**，看今天哪些时段空着，避开已有日程和会议。

3. **提议时间块**：
   - 在空档里排，**块间留 5 分钟缓冲**，别排满（ADHD 友好）。
   - 每块标题加前缀方便一眼认：`[写] lunwen2 intro`、`[读] ElasticMem 实验部分`、`[审] A 方案`、`[专注] <事>`。
   - 把提议**列给用户**：`10:00-10:50 [写] lunwen2 intro ／ 14:00-14:50 [读] ElasticMem …`

4. **确认后才建（硬规则）**：用户确认 / 调整后，用 `lark-calendar` 创建日程（载入 lark-calendar，按它的 `+create` 工作流）。**绝不自动占用户时间**——先预览、确认才写。建完回一句"排好了，X 个块进日历了"。

### B. 补录实际（你干完了告诉我，补到日历 + 时间轴）
用户说"我刚 10-11 写了 lunwen2 intro / 刚才做了 Z"——把**已经发生**的补上：
1. **解析**：时间段 + 做了啥 + 判 domain / 前缀（`[写]/[读]/[审]`）。
2. **建日历块（过去时段）**：用 `lark-calendar` 在那个时间段建日程，标题带前缀。
3. **记时间轴**：`research-assistant timeline-append --tz <tz> --text "10:00-11:00 写 lunwen2 intro ✅"`——这样实际产出也进**周报和 `focus-stats`**。
4. 回执"补好了：日历 + 时间轴各一条"。
> 补录是你**主动报的实际**，直接记 + 回执即可（不像排未来要先提议确认）。

## Pitfalls
- **写日历 = 改你的真实日程**，一律先提议、确认才建，绝不自动占。
- 先查 freebusy 再排，别撞已有日程 / 会议。
- 别排满——块间留 5 分钟缓冲，单块别太长（专注 50 分钟左右一块）。
- **focus-buddy 的"陪我专注"不在这建块**（那是即时会话，不占日历）。
- **补录别和 focus-buddy 重复**：用了"陪我专注"的活已自动进时间轴，别再补录同一段（会重复）。

## Verification
- "帮我把今天排进日历" → 先查忙闲 → 提议带 `[写]/[读]/[审]` 前缀、留 5min 缓冲的块 → 确认后才在飞书日历建出来。
